@Story The ironic thing with that is that I actually have to do that all the time in Objective-C, not because it won't let me use self in a block, but because blocks in Objective-C automatically retain any variables inside of them on initialization. If the block is persistent and I don't special case the pointer to self to not be retained, it never releases the object as long as the block persists (possibly leading to retain loops in the process), causing it to leak.

Really though, having to add one line of code to copy $this isn't that big of a pain, and makes sense due to the semantic meaning of $this (which would have no meaning in an anonymous function).
